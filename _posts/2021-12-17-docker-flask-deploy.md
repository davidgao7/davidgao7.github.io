## 介绍
- `Flask`: python **轻量级** 框架, 符合 WSGI 规范， 不能独立运行（类似app.run的方式仅适合开发模式）， 需要依赖其他的组件提供服务器功能
- `Gunicorn`: unix 上被广泛使用的 **高性能** 的 python WSGI UNIX HTTP Server, 和大多数的 web 框架兼容， 并具有实现 简单、轻量级、高性能 等特点
- `gevent`:  gunicorn 默认使用同步阻塞的网络模型（-k sync）， 并对于**大并发**的访问可能表现**不好**， 我们很方便的顺手套一个 gevent 来 **增加并发量**
- `docker`: 容器， 可以理解成一个 black box， 在项目庞大以后，往往我们会疲于管理整个项目部署维护。 如果我们将整个项目用一个 ‘容器’ 装起来， 那么我们仅仅只维护一个配置文件告诉计算机每次部署要把什么东西装进 '容器' , 甚至借用一些工具把这个过程自动化， 部署就会变的很方便。
